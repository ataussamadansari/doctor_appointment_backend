# 🚀 Setup Checklist - Doctor Appointment System

Complete this checklist to get your system up and running.

---

## ✅ Pre-Setup Requirements

### 1. Software Installation

- [ ] **Node.js 18+** installed
  ```bash
  node --version  # Should show v18.x.x or higher
  ```

- [ ] **npm** installed
  ```bash
  npm --version  # Should show 9.x.x or higher
  ```

- [ ] **Redis** installed
  ```bash
  redis-server --version  # Should show 7.x.x or higher
  ```

- [ ] **Git** installed
  ```bash
  git --version
  ```

## 🌐 MongoDB Atlas Setup (10 minutes)

### 2. Create MongoDB Atlas Account

- [ ] Go to https://cloud.mongodb.com/
- [ ] Sign up with email or Google/GitHub
- [ ] Verify email address

### 3. Create Free Cluster

- [ ] Click "Build a Database"
- [ ] Select "Shared" (FREE)
- [ ] Choose cloud provider and region
- [ ] Click "Create Cluster"
- [ ] Wait 3-5 minutes for cluster creation

### 4. Create Database User

- [ ] Go to "Database Access"
- [ ] Click "Add New Database User"
- [ ] Choose "Password" authentication
- [ ] Set username: `________________`
- [ ] Set password: `________________` (save this!)
- [ ] Select "Read and write to any database"
- [ ] Click "Add User"

### 5. Configure Network Access

- [ ] Go to "Network Access"
- [ ] Click "Add IP Address"
- [ ] Click "Allow Access from Anywhere" (0.0.0.0/0)
- [ ] Click "Confirm"

### 6. Get Connection String

- [ ] Go to "Database" → Click "Connect"
- [ ] Choose "Connect your application"
- [ ] Copy connection string
- [ ] Connection string: `mongodb+srv://________________`

---

## 🔧 Application Setup (5 minutes)

### 7. Clone and Install

- [ ] Navigate to backend directory
  ```bash
  cd backend
  ```

- [ ] Install dependencies
  ```bash
  npm install
  ```

### 8. Environment Configuration

- [ ] Copy environment template
  ```bash
  cp .env.example .env
  ```

- [ ] Open .env file
  ```bash
  nano .env  # or use your preferred editor
  ```

- [ ] Update MongoDB connection string
  ```env
  MONGODB_URI=mongodb+srv://username:password@cluster0.xxxxx.mongodb.net/doctor_appointment?retryWrites=true&w=majority
  ```

- [ ] Replace `username` with your username
- [ ] Replace `password` with your password
- [ ] Replace `cluster0.xxxxx` with your cluster address
- [ ] URL encode password if it has special characters

- [ ] Verify JWT secrets are set (auto-generated by setup.sh)
  ```env
  JWT_SECRET=your-super-secret-jwt-key-min-32-chars
  JWT_REFRESH_SECRET=your-super-secret-refresh-key-min-32-chars
  ```

### 9. Optional: Run Setup Script

- [ ] Make script executable
  ```bash
  chmod +x setup.sh
  ```

- [ ] Run setup script
  ```bash
  ./setup.sh
  ```

## 🚀 Start Services (2 minutes)

### 10. Start Redis

- [ ] Open new terminal
- [ ] Start Redis server
  ```bash
  redis-server
  ```
- [ ] Verify Redis is running
  ```bash
  redis-cli ping  # Should return PONG
  ```

### 11. Start Backend

- [ ] Open new terminal
- [ ] Navigate to backend
  ```bash
  cd backend
  ```

- [ ] Start in development mode
  ```bash
  npm run dev
  ```

- [ ] Check logs for successful startup
  ```
  ✓ MongoDB Connected: cluster0-shard-00-00.xxxxx.mongodb.net
  ✓ Redis connected successfully
  ✓ Socket.io initialized
  ✓ Server running on port 5000
  ```

## ✅ Verification (5 minutes)

### 12. Test Health Endpoint

- [ ] Open browser or use curl
  ```bash
  curl http://localhost:5000/health
  ```

- [ ] Should return:
  ```json
  {
    "success": true,
    "message": "Server is healthy",
    ...
  }
  ```

### 13. Test User Registration

- [ ] Register a test user
  ```bash
  curl -X POST http://localhost:5000/api/v1/auth/register/user \
    -H "Content-Type: application/json" \
    -d '{
      "firstName": "Test",
      "lastName": "User",
      "email": "test@example.com",
      "phone": "9876543210",
      "password": "password123",
      "dateOfBirth": "1990-01-01",
      "gender": "male"
    }'
  ```

- [ ] Should return success with user data and tokens

### 14. Test Login

- [ ] Login with test user
  ```bash
  curl -X POST http://localhost:5000/api/v1/auth/login \
    -H "Content-Type: application/json" \
    -d '{
      "email": "test@example.com",
      "password": "password123"
    }'
  ```

- [ ] Should return success with tokens

### 15. Verify Database

- [ ] Go to MongoDB Atlas dashboard
- [ ] Click on your cluster
- [ ] Click "Browse Collections"
- [ ] Should see `doctor_appointment` database
- [ ] Should see `users` collection with test user

### 16. Update JWT Secrets

- [ ] Generate strong JWT secrets

- [ ] Update in .env file

  JWT_REFRESH_SECRET=<generated-secret-2>

## 🐛 Troubleshooting

### Common Issues

**MongoDB Connection Failed**
- [ ] Check connection string format
- [ ] Verify username and password
- [ ] URL encode password special characters
- [ ] Check IP is whitelisted in Atlas

**Redis Connection Failed**
- [ ] Verify Redis is running: `redis-cli ping`
- [ ] Check REDIS_HOST and REDIS_PORT in .env

**Port Already in Use**
- [ ] Change PORT in .env
- [ ] Or kill process using port 5000

**JWT Errors**
- [ ] Ensure JWT secrets are at least 32 characters
- [ ] Regenerate secrets if needed
