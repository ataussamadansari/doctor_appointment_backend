**Features**:
- ✅ Profile image uploads (auto-optimized, 500x500)
- ✅ Doctor verification document uploads
- ✅ Image optimization (auto quality, format)
- ✅ File validation (type, size)
- ✅ File deletion
- ✅ Public ID extraction from URLs
- ✅ Connection verification on startup

**Supported Formats**:
- Images: JPG, JPEG, PNG, GIF, WEBP
- Documents: PDF, DOC, DOCX

**Size Limits**:
- Images: 5MB
- Documents: 10MB

**API Endpoints**:
```
POST /api/v1/upload/profile-image
  - Upload profile image (patients & doctors)
  - Field name: 'image'
  - Returns: { url, user }

POST /api/v1/upload/verification-document
  - Upload verification document (doctors only)
  - Field name: 'document'
  - Body: { documentType: 'medical_degree' | 'registration_certificate' | ... }
  - Returns: { url, documentType, doctor }

DELETE /api/v1/upload/verification-document/:documentId
  - Delete verification document (doctors only)

GET /api/v1/upload/signed-url/:publicId
  - Get optimized image URL
```

**Usage Example**:
```javascript
// Frontend - Upload profile image
const formData = new FormData();
formData.append('image', file);

const response = await fetch('/api/v1/upload/profile-image', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${token}`
  },
  body: formData
});

// Backend - Upload service
import uploadService from './services/uploadService.js';

const result = await uploadService.uploadProfileImage(base64Image);
// Returns: { url, publicId, format, width, height, size }
```

**Files**:
- `backend/src/config/cloudinary.js` - Cloudinary configuration

- `backend/src/controllers/uploadController.js` - Upload controllers

## 🐛 Troubleshooting

### Email Not Sending
1. Check Gmail App Password is correct
2. Verify "Less secure app access" is enabled (if using regular password)
3. Check firewall/antivirus blocking port 587
4. Review logs for detailed error messages

### Cloudinary Upload Failing
1. Verify API credentials in .env
2. Check file size limits
3. Ensure file format is supported
4. Review Cloudinary dashboard for quota limits

### File Upload Issues
1. Check Content-Type is multipart/form-data
2. Verify field name matches ('image' or 'document')
3. Check file size doesn't exceed limits
4. Ensure Authorization header is present
